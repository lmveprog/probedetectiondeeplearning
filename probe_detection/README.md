# Probe Detection with Deep Learning

Detection of the ultrasonic thickness measurement probe in Elios3 drone
images, based on a fine-tuned YOLO11 model.

**Results on a leakage-free validation split** (76 images, 5 flights never
seen in training — full analysis in `reports/report.pdf`):

| model | precision | recall | F1 | mAP@0.5 | mean IoU | ms/img (CPU) |
|---|---|---|---|---|---|---|
| YOLO11n (recommended) | 0.996 | 0.895 | 0.943 | 0.961 | 0.876 | 18.6 |
| YOLO11s | 0.934 | 0.924 | 0.929 | 0.964 | 0.863 | 33.8 |

The "no probe" confidence threshold (0.30) is calibrated on synthetic
probe-free crops: 92.1% recall at a 3.9% false-alarm rate.

![detection demo](reports/demo/demo_reel.gif)

*Validation flights (never seen in training) replayed from the frame
timestamps embedded in the file names, with EMA-smoothed detections —
generated by `scripts/video_demo.py`.*

## Try it without installing anything

Open `web_demo/probe_demo.html` in any modern browser and drop an image on
it: the trained network itself (ONNX + WebAssembly, single self-contained
file) runs locally — no Python, no upload. Built by
`scripts/build_web_demo.py`.

## Beyond the assignment

- **Flight replay demo** (`scripts/video_demo.py`): the dataset frames are
  timestamped video frames; the demo reconstructs each validation flight and
  runs the detector with the temporal smoothing (EMA + 2-frame hysteresis) a
  real onboard pipeline would use. MP4s in `reports/demo/`.
- **Synthetic negative training data** (`scripts/make_negatives.py`): the
  probe is erased from training frames by inpainting (with re-injected,
  noise-matched sensor grain) and probe-free crops are added, giving 14%
  background images. **Measured outcome: no improvement** — the retrained
  model (`weights/probe_yolo11n_neg.pt`) matches the baseline's recall but
  with a higher false-alarm rate across the operating range
  (`reports/eval_yolo11n_neg/comparison.png`), so the baseline stays the
  delivered model. With only 32 negatives against an already-low baseline
  false-alarm rate, real probe-free flights remain the way to move this
  needle.
- **Robustness study** (`scripts/robustness.py`): recall measured under
  increasing darkness, overexposure, motion blur and sensor noise. Result:
  graceful degradation on the first three, but **sensor noise is the main
  weakness** (recall 0.92 → 0.50 at severity 4) → add noise augmentation /
  light denoising in production. Figure: `reports/eval_yolo11n/robustness.png`.
- **Self-improvement loop** (`scripts/self_training.py`): simulates a
  production feedback system — a weak model (trained on 25% of the flights)
  pseudo-labels the remaining flights *without their human labels*, keeping
  only temporally-stable detections (a probe cannot teleport between frames
  0.3 s apart) above a threshold calibrated on the labeled subset. The
  auto-generated labels reach **IoU 0.86 vs the hidden human annotations**,
  and retraining on them lifts mAP@0.5 from 0.845 to 0.921 — closing ~2/3 of
  the gap to the fully-supervised model **with zero new annotation**.
  Results: `reports/self_training/`.
- **Learning curve** (`scripts/learning_curve.py`): the same recipe trained
  on nested 25/50/75/100% subsets of the training flights (fixed val set).
  Detection presence saturates early (mAP@0.5: 0.951 already at 50% of the
  data) but **localization quality is still climbing at 100%** (mAP@0.5:0.95:
  0.50 → 0.62 → 0.70 → 0.74) — collecting more annotated flights would still
  pay, mostly in box accuracy. Figure: `reports/learning_curve/`.
- **Deployment benchmark** (`scripts/export_bench.py`): ONNX export (the
  graph a Jetson TensorRT engine is built from) + int8 dynamic quantization:
  model shrinks 10.6 → 3.0 MB (3.5×) with detections preserved (conf 0.870 →
  0.866, box center shift 0.1 px). Results: `reports/export_benchmark.json`.

## Quick start — run inference on your validation set

```bash
# 1. create an environment (Python 3.10–3.12) and install the dependencies
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# 2. run the detector on a folder of images
python inference.py /path/to/your/images --output results
```

Each image is processed one by one; the probe bounding box is drawn on it
and the annotated image is saved to the output folder (`results/` above).
If no probe is detected, the image is stamped "NO PROBE DETECTED" and the
absence is reported on the console. A machine-readable `detections.json`
(bbox + confidence per image) is also written next to the images.

Options:

| flag | default | meaning |
|---|---|---|
| `--weights` | `weights/probe_yolo11n.pt` | trained weights to use |
| `--conf` | `0.3` | confidence threshold under which "no probe" is reported (calibrated in `evaluate.py`) |
| `--output` | `output` | folder for annotated images |
| `--show` | off | display each image in a window instead of saving |
| `--device` | auto | `cpu`, `mps`, `cuda:0`, … |

## Reproducing training

```bash
# the original probe_dataset/ folder must sit next to this repository
python scripts/prepare_data.py       # builds data/ with a grouped split by flight
python train.py --model yolo11n.pt   # fine-tunes; results in runs/yolo11n/
python evaluate.py                   # metrics + calibration + runtime figures
```

The train/val split is grouped **by flight** (drone serial + flight counter
parsed from the file names): frames of one flight are near-duplicates, so an
image-level random split would leak into validation and overstate metrics.
